UnifiedAcquire Release Notes Process

Created March 2026  ·  For internal distribution

Overview

Each release cycle, Jira issues tagged with a release label (e.g. UA4.5) are used to generate external- and internal-facing release notes. The notes are drafted using Claude AI, which matches the tone and structure of previous Confluence release notes automatically.

There are two ways to run this process:

Option A — Manual Prompt Free

Use this if you want no infrastructure or API costs. Takes about 2 minutes per release.

Steps

1
Go to claude.ai and start a new conversation. You must have access to the Atlassian MCP integration.
2
Paste the prompt below, replacing the version number and label for the current release.
3
Review the output and post to the UnifiedAcquire Confluence space.

Saved Prompt Template

Create release notes for version [X.X.0] from our Jira board.
Find all issues with the label [UAX.X], then match the tone and
structure of our existing release notes on the UnifiedAcquire
Confluence space (reference the most recent release notes page).
Show me the draft here before posting anywhere.
Example: For the 4.5 release, replace [X.X.0] with 4.5.0 and [UAX.X] with UA4.5.

Option B — Automated Agent Recommended

A Python script runs automatically on the 1st of every month via GitHub Actions. It queries Jira, fetches the latest Confluence release notes for tone reference, generates a draft with Claude, and emails it to you for review before anything is posted.

How It Works

1
Detect version — Finds the latest versioned release notes page on Confluence (e.g. 4.4.0) and computes the next version (4.5.0) and label (UA4.5) automatically.
2
Fetch issues — Queries Jira for all issues tagged with that label. If none are found, the run exits silently with no email.
3
Generate draft — Sends the issues and previous release notes to Claude API, which writes new notes matching the established tone and structure.
4
Email draft — Sends the draft to kevin.jackson@rohirrim.ai for review. You then manually post to Confluence when satisfied.

Running Manually

You can trigger the agent at any time without waiting for the 1st of the month:

  1. Go to your repo on github.com
  2. Click the Actions tab
  3. Click Monthly Release Notes in the left sidebar
  4. Click Run workflow → Run workflow

Jira Labeling Convention

For the agent to pick up issues, they must be labeled in Jira with the correct release label before the 1st of the month when the agent runs.

Release VersionJira Label
4.4.0UA4.4
4.5.0UA4.5
4.6.0UA4.6
5.0.0UA5.0

The agent auto-increments the minor version each month. Major version bumps require manually updating the label format.
